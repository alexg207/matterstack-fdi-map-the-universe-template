# Clay Table Setup — FDI Map the Universe

Step-by-step instructions for building the Clay tables that serve as the raw data source for an FDI intelligence map.

Clay's job is **data gathering only**. All analysis, scoring, copy, signals, tags, detail sections, and GTM strategy are generated by Claude Code after the CSV export.

---

## Table 1: Companies

### Step 1 — Create the table and seed it

1. In Clay, create a new table called `[Client Name] - FDI Companies`
2. Add your seed companies. You need two columns to start:
   - **Company Name** (text) — e.g., "Toshiba"
   - **Domain** (text) — e.g., "toshiba.com"
3. Paste in your 20-30 target companies. These typically come from your initial research or the client brief.

### Step 2 — Add segment and classification columns (manual)

These are columns YOU fill in. They control how the map is organized — which section each company appears in and its priority.

| # | Column Name | Type | How to fill |
|---|-------------|------|-------------|
| 3 | Segment | Single Select | Create one option per group (e.g., "Storage Media", "Photonics / Optical", "Power Semi"). Select one per company. |
| 4 | Segment ID | Text | URL-safe slug matching the segment. "Storage Media" → `storage`, "Photonics / Optical" → `photonics`. Every company in the same segment gets the same ID. |
| 5 | Tier | Single Select | Add three options: `high`, `med`, `low`. Select one per company. High = top priority targets. |
| 6 | Rank | Number | Position within the segment. Most important = `1`, second = `2`, etc. |
| 7 | Rank Total | Number | Total companies in that segment. If Storage Media has 3 companies, every company in that segment gets `3`. |

### Step 3 — Add auto-enriched columns

These use Clay's built-in enrichments. For each one, click **"+"** → **"Enrich data"**, search for the enrichment, map the input to your `Domain` column, and click **Run**.

| # | Column Name | Clay Enrichment | What it returns |
|---|-------------|----------------|-----------------|
| 8 | LinkedIn URL | Company LinkedIn URL | Company LinkedIn page URL |
| 9 | HQ Location | Company HQ Location | City, State, Country |
| 10 | Employee Count | Company Headcount | Current employee count |
| 11 | Annual Revenue | Annual Revenue | Revenue figure or range |
| 12 | Recent News | Recent News | Headlines from last 3-6 months |
| 13 | Competitors | Company Competitors | Top 3-5 competitors |
| 14 | Open Jobs | Open Jobs | Current job postings |

### Step 4 — Add Claygent deep research (single column)

One Claygent column that does a comprehensive pass per company. One credit per company instead of seven.

1. Click **"+"** → **"Use AI"** / **"Claygent"**
2. Name it `Financial & Competitive Research`
3. Paste this prompt:

```
Research {Company Name} ({Domain}) using their most recent 10-K, annual report, and earnings releases. Return the following in this exact format:

REVENUE: [Dollar amount] ([Fiscal year])
R&D SPEND: [Dollar amount] ([Fiscal year]) [Note any multi-year trend if available]
GROSS MARGIN: [Percentage] ([Quarter or fiscal year])
MARKET SHARE: [Percentage] in [segment name] ([Source and timeframe])
COMPETITIVE POSITION: [2-3 sentences on where they stand vs top 3 competitors — who is ahead, behind, and why]
KEY RISKS: [Top 2-3 risk factors from 10-K or annual report]
STRATEGIC MOVES: [Any M&A, partnerships, restructuring, or leadership changes in last 12 months with dates]
GOVT FUNDING: [Any CHIPS Act, grants, or subsidies — amount and purpose, or "None found"]

If any data point is not publicly available, write "Not available" for that line. Be specific with numbers, dates, and sources.
```

4. Make sure `{Company Name}` maps to your Company Name column and `{Domain}` maps to your Domain column
5. Click **Run**

**That's it — 15 columns total for the Companies table.**

## Table 2: Contacts

### Step 1 — Create the table

1. Click **"Create Table"** / **"+ New"**
2. Name it `[Client Name] - FDI Contacts`
3. Choose **"Start from scratch"**

### Step 2 — Add columns

Create these 11 columns in order:

| # | Column Name | Type | How to fill |
|---|-------------|------|-------------|
| 1 | Full Name | Text | Contact's full name |
| 2 | Company Name | Text | Must match the Companies table exactly (e.g., "Toshiba", not "Toshiba Corp") |
| 3 | Job Title | Text | Current title |
| 4 | Contact Type | Single Select | Add options: `technical`, `business`. Technical = engineering/R&D roles. Business = operational/strategic roles. |
| 5 | LinkedIn URL | Text | Profile link |
| 6 | Location | Text | City, Country |
| 7 | Primary Connection | Text | Name of your team member who can intro (e.g., "Alex Giles") |
| 8 | Connection Strength | Single Select | Add options: `warm`, `possible` |
| 9 | Secondary Connection | Text | Optional second intro path |
| 10 | Connection 2 Strength | Single Select | Add options: `warm`, `possible` |
| 11 | Notes | Text | Optional — anything relevant about this contact |

### Step 3 — Find contacts for each company

For each company in your Companies table:

1. Use Clay's **"Find people"** enrichment or manually search LinkedIn
2. Look for 2-5 people with these titles:
   - VP, SVP, EVP, Director, Head of, Chief Technology, Chief Product
   - In these functions: R&D, Engineering, Technology, Product, Manufacturing, Operations
3. **Do NOT add:** CEOs, Chief Executive Officers, Sales, Marketing, HR, Legal
4. Fill in all columns for each contact found

## What Claude Code generates from this data

After you export both tables as CSV, Claude Code reads the raw data and generates everything analytical:

| Generated by Claude Code | Description |
|--------------------------|-------------|
| Subtitle | One-line thesis per company |
| Intelligence Summary | 2-3 sentence overview |
| GTM Thesis | Sales positioning angle |
| Opportunity Score (1-5) | Based on hiring, news, leadership changes from the enriched data |
| Opportunity Reason | One sentence explanation |
| Opportunity Signals | Color-coded signal tags |
| Competitive Distress (1-5) | Based on competitive position research |
| Distress Reason | One sentence explanation |
| Distress Signals | Color-coded signal tags |
| Tags | Colored pills based on tech stack, products, status |
| Detail Sections | Structured data tables (Financial Profile, Competitive Position) |
| Source Citations | Links to 10-Ks, earnings, reports |

## Export and Build

### From Clay
1. In the Companies table, click **Export** → **Download as CSV** → save as `companies.csv`
2. In the Contacts table, click **Export** → **Download as CSV** → save as `contacts.csv`

### In Claude Code
```
Here are two CSVs exported from Clay:
- /path/to/companies.csv
- /path/to/contacts.csv

Generate data.js for the FDI Map the Universe template following
the structure in data.example.js. Analyze the Financial & Competitive
Research column to generate scoring, signals, copy, detail sections,
and source citations for each company.
```

Claude Code will:
1. Read both CSVs
2. Parse the Claygent research output for each company
3. Generate scores, signals, summaries, GTM theses, tags, and detail sections
4. Group companies by Segment
5. Map contacts to their companies
6. Write `data.js`
7. Open the map in browser for review

## Workflow Summary

| Phase | Time | What |
|-------|------|------|
| 1. Seed | 10 min | Paste company names + domains into Clay |
| 2. Classify | 10 min | Assign segments, tiers, and ranks |
| 3. Auto-enrich | 5 min | Run Clay enrichments (revenue, news, headcount, competitors, jobs) |
| 4. Deep research | 5 min | Run the single Claygent column |
| 5. Find contacts | 20 min | Use Clay people search per company, assign connection paths |
| 6. Export | 2 min | Download both CSVs |
| 7. Claude Code | 10 min | Generate data.js, review output, iterate on copy |
| **Total** | **~1 hour** | From company list to live intelligence map |

## Tips

- **Start with the demo sections.** If you're presenting 2-3 segments, polish those first. The rest can be thinner.
- **Let Claude Code do the writing.** Your job in Clay is to gather accurate raw data. Claude Code turns it into polished copy.
- **Review Claude Code's output.** After it generates the map, walk through each company and tell it what to fix. This is faster than writing from scratch.
- **Connection mapping is the last step.** Don't spend time on it until the company data is solid.
- **The Claygent prompt is the most important piece.** If the research output is thin, the downstream analysis will be thin. Tweak the prompt if needed for your specific industry.
